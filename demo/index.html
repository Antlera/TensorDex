Tensor-aware storage for the weights you keep re-saving.

The catch. A delta only pays off between similar tensors — so first you must find, across thousands of them, which pairs are close. Reading the real weights to check is the expensive part.
Why a sketch

Measure similarity without ever reading the weights.

Pairwise similarity over N tensors is O(N²) — ruinous over megabytes each. TensorSketch collapses every tensor to a fixed 2 KB sketch whose L2 distance estimates the originals' Hamming distance — which a fitted model maps straight to a predicted compression ratio.

So why not just measure similarity directly?

option A

Actually compress every pair

Run the real codec on every candidate pair — i.e. do the very work you're trying to plan, N² times over.

≈ N² codec runs · circular
option B

Compare the full weights

Diff raw bytes pairwise. Correct — but every pair reads two megabyte-scale tensors; bandwidth dies long before the table is scored.

≈ N² × MBs · bandwidth-bound
option C — TensorDex

Compare the sketches

Hash each tensor to a 2 KB sketch once at ingest. Every pair is then a tiny-vector L2 → predicted CR — the weights are never touched again.

≈ N² × 2 KB · the cheap one
The plan · one idea, two stores
FlexSplit model store

Across a collection of models — e.g. 48 fine-tunes of one base. Everything starts attached to a single base (a star); FlexSplit then splits off a new base wherever a candidate earns more than it costs. This reaches the 27.4%.

same idea, now on a time-ordered checkpoint chain ↓
FlexSplit checkpoint store

One training run emits a time-ordered chain of checkpoints. Every pair has a predicted ratio, so the plan picks the topology — which checkpoints stay raw bases, which attach as deltas. FlexSplit recursively divides the series, opening a new base only where it pays off; the fingerprints decide how many, not a fixed K.

Interactive — drift a checkpoint run, watch FlexSplit decide

Bars are predicted storage (TensorSketch → Hybrid CR model); the real _split recursion runs live, ported to JS.

What the plan buys you
  • Free to plan. The layout is chosen from 2 KB sketches — zero actual compression to decide it.
  • Adaptive. The data picks the number of bases; no fixed K to guess, no bases wasted on the smooth stretch.
  • Depth 1. Every member deltas to a raw base — one hop to reconstruct, unlike a depth-N chain.
  • Global. It opens a base exactly where the series diverges, beating star / k-center / chain on the storage–depth trade-off.
