Tensor-centric storage for AI model hubs · Rust + Python

Store your model hub in 29% of the space.

TensorDex manages model storage with the tensor as the first-class unit, not the whole model — deduplicating identical tensors across models and delta-compressing the rest. On a real trace of 2,890 HuggingFace models it cuts total storage by 70.5% — a 3.39× reduction, and 37% below the previous state of the art — while reconstructing byte-exact .safetensors on demand.

Why the tensor is the right unit

Redundancy lives at the tensor level — so storage should be managed there too.

By 2025, fine-tuned derivatives are 99.1% of HuggingFace storage and 99.6% of all models. These variants share most of their tensors but each occupies a full model's footprint. Manage storage one model at a time and that overlap is invisible; make the tensor the first-class unit and it collapses — identical tensors stored once, the rest as deltas against the closest base anywhere in the hub.

The hard part. A delta only pays off between similar tensors — so first you must find, across millions of them, which pairs are close, without lineage metadata and without reading every byte. That's what the next two ideas solve.
How it works · one sketch, two planners

Find redundancy for free — then save your models and your checkpoints.

foundation

TensorSketch

A compact bit-level fingerprint per tensor. A tiny 2 KB sketch stands in for a megabyte-scale weight — its distance predicts how well two tensors delta-compress, so similarity is found without ever reading the weights.

planner · models

FlexSplit

Organizes a whole collection of models into multi-center clusters, splitting off a new base wherever it pays for itself. This is what reaches 70.5% across 2,890 models.

planner · checkpoints

FlexSplit

Recursively divides a time-ordered checkpoint chain, opening a raw base only where the run diverges — every checkpoint one hop from a base, reconstructed byte-exact.

Both planners rest on the same trick — comparing N tensors is O(N²), so a sketch is the only way it scales:

Lossless isn't a claim — reconstruct a tensor and check the hash yourself.

No mockups. This stores a target tensor as a delta against a base, reconstructs it base + delta, verifies the result is byte-identical with SHA-256, and writes a valid .safetensors file you can open with the safetensors library.

Under the hood

Python orchestrates. Rust owns the bytes and the truth.

Python never holds durable state — SQLite is the single source of truth. The sketch and every codec live in Rust, so ingest is one atomic call over memory-mapped safetensors.
