# TensorDex — Results

This document mirrors the paper's evaluation. The figures below are pre-rendered
(committed so they show on GitHub); regenerate them yourself from the cache — the
images update in place, nothing else to do:

```bash
make ae-cache-figures   # download the cache (results.db + aux; skips the blobs)
make figures            # → ae/figures/*.png   (overwrites the images below)
```

(Or `make ae-cache && make reproduce-all` to regenerate every figure **and**
run all the verification experiments in one command, ~30–45 min.)

For a browser view, `make report` builds a self-contained, single-column
`ae/results.html` (figures embedded — one portable file), and `make serve` hosts
it on `localhost:8000` (expose publicly with `cloudflared tunnel --url
http://localhost:8000`, or drop `results.html` on Cloudflare Pages).

Each result also lists how to *re-derive* it from raw data (not just re-plot).
See [FIGURE_MAP.md](FIGURE_MAP.md) for the full figure↔chart table and
[README.md](README.md) for the three reproducibility tiers.

## Headline (Fig 1)

TensorDex stores a randomly-sampled Hugging Face corpus in **0.29×** the space
(70.5 % saved), well below every baseline, at high throughput.

![Fig 1 left — normalized storage reduction](figures/codec_storage_reduction.png)

![Fig 1 right — compression/decompression throughput](figures/codec_throughput.png)

*Reproduce the reduction:* `make verify` re-derives the TensorX and FM++
per-tensor ratios bit-exact from raw bytes. *Throughput* is hardware-measured
(§6.2.2).

## TensorSketch & prediction (§6.3, Fig 12–13)

### Recall@1 — sketch-based base selection matches exact search (Fig 12a)
![Fig 12a — Recall@1](figures/bcs_recall.png)

TensorSketch + HNSW selects the **same delta base** as exact brute-force search —
Recall@1 = 1.00 — from 8 KB fingerprints instead of full tensors.

*Reproduce:* `make verify-recall` — greedy base-selection with brute-force vs
HNSW indexes over recomputed BCS fingerprints; reports Recall@1 = 1.00, offline.

### Reduction-ratio prediction accuracy (Fig 13)
![Fig 13 — predicted vs real reduction ratio](figures/pred_vs_real_ratio.png)

TensorPred predicts per-tensor reduction from sketch distance with mean error
~1 %, without materializing any delta. Reproduced as an experiment
(`make verify-predict`): the hybrid model is re-fit from the cached ratios by
least squares and evaluated on a held-out split — **MAE 1.11 %, Pearson 99.3 %**,
matching the paper with no pre-recorded coefficients.
